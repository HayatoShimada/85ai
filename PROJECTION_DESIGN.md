# 演出画面 設計書 - 85-Store AI店員システム

## 概要

店舗内のプロジェクターに投影する演出専用画面の設計です。
メインのタッチスクリーン（iPad等）とは別デバイスで、壁面投影用の映像をブラウザ上で生成します。

**技術的な仕組み:**
- iPad操作画面 (`/`) → WebSocket (`/ws/projection/control`) → バックエンド → WebSocket (`/ws/projection/display`) → 演出画面 (`/projection`)
- 別デバイス・別ブラウザ間でも動作（WebSocket経由のため同一ネットワーク内なら可）
- ミラー映像（人物セグメンテーション）も同じWebSocket接続でリアルタイム配信

---

## 画面遷移フロー

```
IDLE（待機） → PREFERENCE（好み入力） → CAMERA_ACTIVE（撮影準備）
     ↑                                          ↓
     └────────── RESULT（結果表示） ←── ANALYZING（解析中）
```

各状態の切り替えはiPad操作画面で発生し、WebSocket経由で演出画面に伝達される。

---

## Scene 1: IDLE（待機・アトラクトモード）

**目的:** 来店客の注意を引き、システムの存在を認知させる

### 実装内容
- 暗いグラデーション背景（slate→emerald→cyan）がゆっくり変化
- 「VINTAGE.AI」ロゴが大きくパルス（拡縮）アニメーション
- 「AIがあなたにぴったりの一点モノを見つけます」テキストがフェードイン/アウト
- 12個の光のパーティクルが浮遊（固定座標、SSRハイドレーション対応済み）
- 「タッチスクリーンからスタートしてください」案内テキスト

### 演出の方向性
- **色調:** ダークベース（#0a0f1a付近）に、エメラルドグリーン(#10B981)とシアン(#06B6D4)のアクセント
- **雰囲気:** ヴィンテージ感 x テクノロジー。古着屋の温かみとAIの先進性の融合
- **動き:** 穏やかで引き込まれるような動き。急激な変化は避ける

---

## Scene 2: PREFERENCE（好み入力中）

**目的:** 来店客が好みを入力している間の間を持たせる雰囲気づくり

### 実装内容
- 背景がインディゴ系に変化（IDLEとは異なるムード）
- 「好みを入力中...」テキスト表示
- お客様名が入力されると「ようこそ、○○さん」表示
- 選択されたスタイルタグがリアルタイムでポップインアニメーション

### 受信データ
| データ | 内容 | 例 |
|--------|------|-----|
| `userName` | お客様名 | 「田中太郎」 |
| `selectedTags` | 選択中の好みタグ配列 | `["かっこいい", "ストリート", "90s"]` |

---

## Scene 3: CAMERA_ACTIVE（撮影スタンバイ）

**目的:** 撮影の準備ができていることを視覚的に伝える。カメラの前に立つよう促す。

### 実装内容
- 暗い背景にグリーンのグリッドパターン（サイバーパンク風スキャンUI）
- 横方向のスキャンラインが上から下へ繰り返しスイープ
- カメラのビューファインダー風の四隅ブラケット
- 中央に「撮影スタンバイ」テキスト
- 画面下部に選択済みタグを表示
- **ミラーオーバーレイ**: 人物セグメンテーション映像を前面に重畳表示（z-index: 30）

### ミラー映像の自動制御
- `ProjectionManager` が `CAMERA_ACTIVE` 状態を検知 → ミラーカメラ自動起動
- OpenCV + MediaPipe で人物切り抜き → base64 PNG → WebSocket配信
- プロジェクター画面で来店客のシルエットがリアルタイムで投影される

---

## Scene 4: ANALYZING（AI解析中）

**目的:** AI処理の待ち時間（10〜30秒）を飽きさせない演出。ワクワク感を醸成。

### 実装内容
- 撮影した画像を大きく中央表示（画面の40%程度）
- 画像の上を緑のスキャンラインが繰り返し通過
- 画像の周囲に回転する点線リング（2重、逆回転）
- 画面左右に「データストリーム」（カタカナ文字が上から下に流れるマトリックス風、固定シード）
- 「AIがコーディネートを考案中...」テキストが明滅
- 30秒超過時: 「少々お待ちください...」メッセージ表示
- **ミラーオーバーレイ**: CAMERA_ACTIVE から継続して表示

### 受信データ
| データ | 内容 |
|--------|------|
| `capturedImage` | 撮影画像（base64 data URL） |
| `analyzeTimedOut` | 30秒超過フラグ |

---

## Scene 5: RESULT（結果表示）

**目的:** AI提案の結果を大画面で見やすく表示。商品のQRコードでスマホ購入を促す。

### 実装内容
- 撮影画像 + バウンディングボックス（AIが認識した服の領域）を左上に表示
- AI分析テキストを大きなフォントで表示
- 検出スタイルタグ + ユーザー選択好みタグを表示
- 3カラムグリッドで最大3パターンの提案を表示
  - 各パターン: タイトル、提案理由、検索キーワード
  - 各パターンに最大2商品: 商品画像、商品名、価格、QRコード（SVG直接表示）
- QRコードは各商品のShopify URLにリンク → スマホで読み取ってEC購入

### 受信データ
| データ | 内容 |
|--------|------|
| `capturedImage` | 撮影画像 |
| `recommendation.analyzed_outfit` | AI分析テキスト |
| `recommendation.detected_style` | 推測スタイルタグ |
| `recommendation.box_*` | バウンディングボックス座標 (0-1000) |
| `recommendation.recommendations[]` | 提案パターン配列（最大3） |
| `selectedTags` | ユーザーが選択した好みタグ |

---

## フラッシュ演出

撮影ボタン押下時にiPadから `FLASH` メッセージが送信される。
プロジェクター画面で白い全画面オーバーレイを表示（1.2秒でフェードアウト）。
フラッシュ後300msの遅延で撮影を実行（プロジェクターの白画面が照明として機能）。

---

## 共通ビジュアルガイドライン

### カラーパレット
| 用途 | カラー | コード |
|------|--------|--------|
| 背景（ベース） | ダークネイビー | `#020617` ~ `#0f172a` |
| アクセント（メイン） | エメラルドグリーン | `#10B981` |
| アクセント（サブ） | シアンブルー | `#06B6D4` |
| テキスト | スレートホワイト | `#f1f5f9` |
| サブテキスト | スレートグレー | `#94a3b8` |
| 警告 | アンバー | `#f59e0b` |
| エラー | レッド | `#ef4444` |

### アニメーション原則
- **トランジション:** 各シーン間は0.5〜1秒のフェード切り替え
- **ループアニメーション:** 穏やかで催眠的な動き。3〜8秒サイクル
- **リアクティブ:** データの変化（タグ追加等）にはスプリングアニメーションで反応
- **FPS:** 60fps維持。CSS/SVGアニメーションを優先（GPUアクセラレーション）
- **SSR対応:** `Math.random()` は使用せず固定シード値を使用（ハイドレーション不一致防止）

## 技術仕様

### 対応解像度
- 推奨: 1920x1080 (Full HD)
- 対応: 1280x720 ~ 3840x2160
- アスペクト比: 16:9 前提（ブラウザのフルスクリーンで表示）

### 実行環境
- ブラウザ: Chrome（推奨）、Edge
- フレームワーク: Next.js 16 + React 19
- アニメーション: Framer Motion + CSS Animations
- QRコード: qrcode.react (SVG出力)
- 状態同期: WebSocket (`/ws/projection/display`)

### ファイル構成
```
frontend/src/
├── app/
│   ├── page.tsx              # iPad操作画面（WebSocket送信側）
│   └── projection/
│       └── page.tsx          # 演出画面（WebSocket受信側）
├── components/
│   └── QRCode.tsx            # QRコードモーダルコンポーネント
└── lib/
    └── projection-types.ts   # 共有型定義（AppState, ProjectionPayload等）
```

### WebSocketメッセージ形式

**JSON メッセージ:**
```typescript
// 状態変更
{ type: "STATE_CHANGE", state: AppState, payload: ProjectionPayload }

// フラッシュ（撮影時）
{ type: "FLASH" }

// 状態要求（接続時）
{ type: "REQUEST_STATE" }
```

**テキストメッセージ（ミラーフレーム）:**
- `{` で始まらない文字列 = base64エンコードされた透過PNG
- `CAMERA_ACTIVE` / `ANALYZING` 状態でのみ配信

### 映像素材の組み込み方法

1. **動画背景:** `public/projection/` に配置 → `<video>` タグで背景に使用
2. **画像テクスチャ:** `public/projection/textures/` に配置 → CSS background
3. **Lottieアニメーション:** `public/projection/lottie/` に配置 → lottie-react で再生
4. **シェーダー:** 複雑なエフェクトは Three.js / WebGL で実装可能
