# 改善計画

## 1. 採寸データの正規化 + 体型情報反映 (優先度: 高)

### 現状の問題

体型情報（身長・体重・肩幅・胸囲・ウエスト）をGeminiプロンプトに渡しているが、
**カタログTSVに商品の実寸サイズが含まれていない**ため、Geminiがサイズマッチングできない。

### 調査結果 (2026-03-10)

**196件中191件（97%）で採寸データの自動抽出に成功。**

#### フォーマット分類

| タイプ | フォーマット | 件数 | 例 |
|--|--|--|--|
| トップス(古着) | `肩幅49cm - 身幅63cm - 袖丈62cm - 着丈82cm` | ~170件 | 一点もの、実寸1つ |
| トップス(新品) | `着丈 約68cm \| 身幅 約68cm \| 肩幅 約66cm` | ~6件 | ONE SIZE、「約」付き |
| ボトムス(古着) | `ウエスト 82cm- 股下 78.5cm- もも周り 35cm` | ~5件 | 一点もの、実寸1つ |
| ボトムス(新品/複数サイズ) | テーブル形式 `ウエスト 股上 股下 ... size1(S) 56cm～ ...` | 3件 | S/M等の複数サイズ |
| 採寸なし | 説明文に採寸データなし | 5件 | 書籍1、刺繍スウェット2、S/M目安のみ2 |

#### 抽出済みスクリプト

`backend/normalize_measurements.py` 作成済み:
- ドライラン: `python3 normalize_measurements.py`
- 適用: `python3 normalize_measurements.py --apply`
- Shopify メタフィールド `custom:measurements` (JSON) に保存

#### メタフィールド保存形式

```json
// トップス（一点もの）
{"肩幅": 49, "身幅": 63, "袖丈": 62, "着丈": 82}

// ボトムス（一点もの）
{"ウエスト": 82, "股上": 34, "股下": 78.5, "もも周り": 35, "裾周り": 22}

// 新品（複数サイズ）
{"sizes": {"S": {"ウエスト": 56, "股上": 44, "股下": 52, "ワタリ": 36, "裾幅": 21, "総丈": 91},
           "M": {"ウエスト": 62, "股上": 47, "股下": 55, "ワタリ": 38, "裾幅": 23, "総丈": 96}}}
```

### 実装ステップ

#### Step 1: メタフィールド書き込み (normalize_measurements.py)
- [x] スクリプト作成済み
- [ ] `--apply` でShopifyに書き込み実行
- [ ] 新品のテーブル形式パーサーの検証・改善（着丈前/着丈後の2列パターンへの対応）

#### Step 2: カタログTSVに採寸列を追加 (catalog_service.py)

`_build_compact_line()` を改修し、メタフィールド `custom:measurements` を読み込んでTSVに追加:

```
ID  Type  Title                              Attrs         Size
0   USED  [Claiborne] Linen BD Shirts L/S    Claiborne     肩幅47,身幅57,袖丈67,着丈83
95  NEW   [River] Wool Check Pants [BROWN]   River,Wool    S:W56/股下52 M:W62/股下55
```

変更箇所:
1. GraphQL クエリに `measurementsMf: metafield(namespace: "custom", key: "measurements") { value }` を追加
2. `_build_compact_line()` で採寸データをコンパクト文字列化して末尾に追加
3. トークン増加量: 約 15文字/商品 × 196件 ≈ 3,000文字（許容範囲）

#### Step 3: Geminiプロンプトにサイズマッチング指示を強化 (gemini_service.py)

```python
body_section = f"""
【ユーザー体型】
{', '.join(parts)}

【サイズマッチングルール】
- カタログのSize列に実寸（肩幅・身幅・着丈等）がある商品は、ユーザー体型との適合度を考慮
- トップス: 肩幅はユーザー+3〜+8cmがジャスト、身幅はユーザー胸囲/2+5〜+15cmが標準
- ボトムス: ウエストはユーザー+2〜+5cmがジャスト
- 複数サイズ(sizes)がある商品は最適サイズを選び、提案理由にサイズを明記
- サイズが合わない商品はproduct_idsに含めない
- 提案理由にサイズフィットについて具体的に言及すること（例: 「肩幅48cmで、お客様の肩幅に+5cmのゆとり」）
"""
```

#### Step 4: フロントエンドに体型入力UIを追加

`PreferenceView.tsx` に体型入力フォームを追加:
- 身長 (cm) — 数値入力
- 体重 (kg) — 数値入力 (任意)
- 肩幅 (cm) — 数値入力
- 胸囲 (cm) — 数値入力
- ウエスト (cm) — 数値入力

入力値は `customer_service.py` 経由で Shopify メタフィールド `custom:body_measurements` に保存。
リピーター客はメール入力で復元。

### 効果の検証

改修後、以下を確認:
1. 小柄な体型（身長160cm, 肩幅40cm）→ 身幅50cm台の商品が提案されるか
2. 大柄な体型（身長185cm, 肩幅50cm）→ 身幅60cm以上の商品が提案されるか
3. 新品で複数サイズがある商品 → 適切なサイズが提案理由に明記されるか
4. 提案理由にサイズフィットの言及があるか

### 注意点

- 元データの誤記あり（例: `袖丈555cm`）→ 閾値チェック（>200cmは異常値として除外）の追加を検討
- 新品のテーブル形式で `着丈前`/`着丈後` の2列を持つパターンがある → パーサー要拡張
- 新品で今後サイズバリエーションが増える場合、Shopifyバリアントにサイズを持たせる運用への移行も検討
