# 次世代古着屋AI店員デバイス 設計ドキュメント

## 1. プロジェクト概要
実店舗（オフライン）とネットショップ（Shopify）のデータ、そして最新のAI（Gemini）を融合させた「次世代の古着屋体験」を提供するAI接客デバイス。
お客様がその日に着ている服をカメラで読み取り、AIがそれに合う「一点モノ」の古着をShopifyのリアルタイム在庫から提案する。

## 2. 全体のシステム構成と処理フロー
画像や音声の認識系（カメラ・マイク）はホストPC（Mac Studio）に直接接続して処理を行い、iPadはリモート操作用のUIおよびストリーミング端末として機能する。プロジェクターによる空間演出も並行して行う。

1. **待機・起動 (iPad / Mac Studio):** iPadブラウザ上のUIで「タップしてAIに服を選んでもらう」などの操作を行い、Mac Studio側の処理をトリガーする。
2. **入力 (Mac Studio接続のカメラ・マイク):** Mac Studioに接続された高性能カメラとマイクでお客様を撮影・録音する。iPad側にはその映像をストリーミング表示して確認可能にする。
3. **AI解析 (バックエンド/Mac Studio):** Mac Studioから Gemini へ画像と音声を渡し、「今着ている服の特徴」と「それに合う服の条件（検索タグ例: `90s, オーバーサイズ, デニムジャケット`）」を推論させる。
4. **データ検索 (バックエンド/Mac Studio):** Geminiが生成した検索キーワードを使って、Shopify Storefront API を叩き、**「現在在庫がある商品」**の中からマッチするものを検索する。
5. **出力 (iPad & プロジェクター):** 見つかった商品の詳細や購入用QRコードをiPadに表示させる。同時に、**店舗の壁一面にプロジェクター（Mac Studio接続）から対象商品のビジュアルや映像演出をプロジェクションマッピング投影**する。スピーカーから合成音声で提案も行う。

## 3. ハードウェア構成
手元の直感的な操作パネル・ストリーミング端末（iPad）、高度な認識・映像処理を行う強力なホスト（Mac Studio）、そして空間演出（プロジェクター）を組み合わせた構成。

* **操作・接客UIデバイス: iPad**
  * お客様・スタッフが手元で操作・商品閲覧を行うためのメイン端末。Mac Studioから配信されるカメラ映像のストリーミング表示や、接客UIレイヤーとして機能するブラウザアプリを稼働させる。
* **ホストPC (認識・映像演出・バックエンド): Mac Studio**
  * カメラやマイクを直接接続し、映像・音声データの取得とローカル処理を行う。非常に高いグラフィック性能と処理能力を活かし、同時にプロジェクションマッピング映像の生成・プロジェクターへの出力、必要に応じたクラウド通信を担う中核ハブとなる。
* **カメラ＆マイク: Webカメラ / 全指向性マイク (Mac Studioへ接続)**
  * 高画質・高音質でのデータ取得用。iPadに頼らず、Mac Studio側に設置された固定カメラ等を活用する。
* **プロジェクター: 超短焦点プロジェクター または 高輝度プロジェクター (3000〜5000ルーメン以上推奨)**
  * Mac Studioから壁一面へダイナミックな投影用。店舗スペースの制約を考慮し、至近距離から大画面を投影できる超短焦点モデルを想定。
* **スピーカー: 店舗用スピーカーシステム**
  * Mac Studioから出力。AI音声の出力や、映像演出に合わせた効果音・BGMを流すために使用。

## 4. ソフトウェア・技術スタック
映像の取得・認識とプロジェクションマッピングの重い処理はMac Studioに寄せ、iPadはWebブラウザで軽量にUIストリーミングを行う。

* **フロントエンド (iPad Webアプリ): `React (or Next.js) + Tailwind CSS`**
  * 直感的で美しいUIの構築。Mac Studioからのカメラ映像ストリーム（WebRTCやMJPEG等）の受信・表示や、WebSocketを利用したリモコン操作を行うUI画面を提供する。
* **映像・認識処理コア (Mac Studio側): `Python (OpenCV, FastAPI) + TouchDesigner / p5.js 等`**
  * Mac Studioに接続したカメラ映像をキャプチャし、AI連携・データ処理を行うPythonベースのバックエンドを稼働。同時にプロジェクターへの映像生成（TouchDesigner等）へデータを流す。必要に応じてクラウドAPIも利用する。
* **AI・推論エンジン: `Google Gemini API (gemini-2.5-flash)`**
  * 画像と音声のマルチモーダル処理。実店舗での接客スピードを重視し、レスポンスが高速な最新Flashモデルを採用。
* **データベース＆EC連携: `Shopify Storefront API (GraphQL)`**
  * リアルタイムな在庫状況（`availableForSale: true`）と商品情報の取得に使用。

## 5. 実現のための重要な技術ポイント

### ① Geminiへのプロンプト設計
ただ画像を投げるだけでなく、システムが扱いやすいデータ形式（JSON形式など）で出力させる。
```json
{
  "reason": "今のパンツが太めのデニムなので、オーバーサイズのカレッジロゴスウェットでアメカジ風に合わせるのがおすすめです。",
  "search_keywords": ["スウェット", "オーバーサイズ", "ネイビー", "90s"],
  "category": "トップス"
}
```

### ② 商品データの「タグ付け」
Shopify側に商品登録する際、AIが検索しやすいように「色（ネイビー）」「柄（チェック）」「年代（80s）」「スタイル（ストリート）」などのタグ（Tags）やメタフィールドをしっかり入力しておくことが、精度の高い提案の鍵となる。

### ③ RAG（検索拡張生成）の導入検討
キーワード検索だけではマッチしづらい場合、Shopifyの全商品データをあらかじめテキスト化してベクトルDBに入れ、Geminiに「現在の在庫リストの中から最も合うものを選んで」と指示する仕組み（RAG）を将来的に検討する。

## 6. 開発のステップ

1. **STEP 1: PC上でプロトタイプ版を作成**
   * PythonやNode.jsを使い、「自分の写真をGemini APIに送る」→「Shopify APIからキーワードで商品情報が返ってくる」というコア機能のプログラムを作成する。
2. **STEP 2: RAGの検証・評価**
   * 検索精度を高めるため、ベクトルDBの導入とRAGアプローチをテストする。
3. **STEP 3: 映像ストリーミングとUI開発**
   * Mac StudioからiPadへの映像ストリーミング基盤を構築し、Reactで操作用UIを作成して結合する。
4. **STEP 4: デバイス連携と店舗空間への実装**
   * Mac Studioにカメラ・マイク・プロジェクターを接続して機能を確認。iPadをリモコン代わりにして「壁面映像の切り替え」や「接客UI」が連動する実環境テストを行う。
