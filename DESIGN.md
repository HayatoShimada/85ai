# 次世代古着屋AI店員デバイス 設計ドキュメント

## 1. プロジェクト概要
実店舗（オフライン）とネットショップ（Shopify）のデータ、そして最新のAI（Gemini）を融合させた「次世代の古着屋体験」を提供するAI接客デバイス。
お客様がその日に着ている服をカメラで読み取り、AIがそれに合う「一点モノ」の古着をShopifyのリアルタイム在庫から提案する。

## 2. 全体のシステム構成と処理フロー
ラズパイ（デバイス側）は「目と耳と口」として働き、重い処理やデータ検索はバックエンド（クラウド等）で行う構成とする。

1. **待機・起動 (デバイス):** ディスプレイに「タップしてAIに服を選んでもらう」などのスタートボタンを表示。お客様がタッチパネルでボタンを押すことで処理が開始される。
2. **入力 (デバイス):** ウェブカメラで5秒後に、お客様を撮影。マイクで「このパンツに合う服を探して」などの音声リクエストを受け取る。
3. **AI解析 (バックエンド):** 撮影した画像と音声を Gemini に渡し、「今着ている服の特徴」と「それに合う服の条件（検索タグ例: `90s, オーバーサイズ, デニムジャケット`）」を推論させる。
4. **データ検索 (バックエンド):** Geminiが生成した検索キーワードを使って、Shopify Storefront API を叩き、**「現在在庫がある商品」**の中からマッチするものを検索する。
5. **出力 (デバイス):** 見つかった商品の画像や値段、購入用QRコードをディスプレイに表示し、スピーカーから合成音声で「こんなスウェットはいかがですか？」と複数商品を提案する。

## 3. ハードウェア構成
店舗に置く「キオスク端末（専用操作パネル）」として機能させるための構成。

* **メインボード: Raspberry Pi 5 (8GBモデル)**
  * AI APIとの通信、リッチなUI表示、画像処理をスムーズに行うため、最新かつメモリに余裕のあるモデルを選定。
* **カメラ＆マイク: Logicool C920n**
  * フルHDの高画質で、「服の素材感や色味」をAIに正確に認識させる。内蔵マイクで音声も取得する。
* **スピーカー: 小型PCスピーカー または 有線接続スピーカー**
  * 音声提案用。遅延や接続トラブルを防ぐため、BluetoothではなくUSBや有線接続を推奨。
* **ディスプレイ: 10〜15インチ タッチ対応モバイルモニター**
  * お客様がスワイプで商品を見たり、QRコードを表示するためタッチパネルが最適。

## 4. ソフトウェア・技術スタック
ラズパイ上で重い処理はせず、UI表示に特化させ、バックエンドはモダンなWeb技術で構築する。

* **フロントエンド (ラズパイ側): `React (or Next.js) + Tailwind CSS`**
  * 直感的で美しいUIを構築。Chromiumブラウザのキオスクモードを用いて全画面表示する。WebRTCでカメラ撮影を実装。
* **バックエンド (クラウド or ラズパイ上): `Python + FastAPI`**
  * AI連携とデータ処理を担当。FastAPIにより高速なリクエスト処理を実現。
* **AI・推論エンジン: `Google Gemini API (gemini-2.5-flash)`**
  * 画像と音声のマルチモーダル処理。実店舗での接客スピードを重視し、レスポンスが高速な最新Flashモデルを採用。
* **データベース＆EC連携: `Shopify Storefront API (GraphQL)`**
  * リアルタイムな在庫状況（`availableForSale: true`）と商品情報の取得に使用。

## 5. 実現のための重要な技術ポイント

### ① Geminiへのプロンプト設計
ただ画像を投げるだけでなく、システムが扱いやすいデータ形式（JSON形式など）で出力させる。
```json
{
  "reason": "今のパンツが太めのデニムなので、オーバーサイズのカレッジロゴスウェットでアメカジ風に合わせるのがおすすめです。",
  "search_keywords": ["スウェット", "オーバーサイズ", "ネイビー", "90s"],
  "category": "トップス"
}
```

### ② 商品データの「タグ付け」
Shopify側に商品登録する際、AIが検索しやすいように「色（ネイビー）」「柄（チェック）」「年代（80s）」「スタイル（ストリート）」などのタグ（Tags）やメタフィールドをしっかり入力しておくことが、精度の高い提案の鍵となる。

### ③ RAG（検索拡張生成）の導入検討
キーワード検索だけではマッチしづらい場合、Shopifyの全商品データをあらかじめテキスト化してベクトルDBに入れ、Geminiに「現在の在庫リストの中から最も合うものを選んで」と指示する仕組み（RAG）を将来的に検討する。

## 6. 開発のステップ

1. **STEP 1: PC上でプロトタイプ版を作成**
   * PythonやNode.jsを使い、「自分の写真をGemini APIに送る」→「Shopify APIからキーワードで商品情報が返ってくる」というコア機能のプログラムを作成する。
2. **STEP 2: RAGの検証・評価**
   * 検索精度を高めるため、ベクトルDBの導入とRAGアプローチをテストする。
3. **STEP 3: Web API化とフロントエンド開発**
   * バックエンド処理をFastAPIでAPI化し、ReactでUIを作成して結合する。
4. **STEP 4: ラズパイへの実装**
   * ラズパイにカメラ・モニターを設定し、キオスクモードとしてWebアプリを稼働させる。
