# UI と AI

コンテクストが好きです。
その決定に至るまでの仮定と、フィードバック、反発と肯定を経て、振り子のように収束していくプロセスは美しい。

UIデザインも同じで

```
装飾過多  →  バウハウス（機能主義）
無機質    →  ポストモダン（装飾の復権）
スキュアモーフ →  フラット
フラット過ぎ  →  マテリアル（奥行きの復権）
マテリアル   →  ??
```

常にその時代の技術と美意識・ユーザビリティのバランスで揺れ動いてきました。

## スーザン・ケア

初めてPCに「顔」を与えたAppleのグラフィックデザイナー"スーザン・ケア"は、CLI中心だったこれまでのPCにGUIを導入するにあたり、当時のグラフィック性能の限界の中で、ユーザーに歩み寄る姿勢を示しました。

技術が進歩していく中で、ドット絵だった衣裳から、質感を感じさせる初代iPhoneのアプリアイコンのような、まるで触れると凹みそうなスキュアモーフィックデザインへと移行しました。これはケアの仕事の延長線上にあります。「コンピュータは難しくない。あなたの知っているものと同じだよ」という翻訳の試みが、より精巧になっていった時代です。

しかしその後、Jony IveがiOS 7でスキュアモーフィズムを一夜にして葬り去ったとき、何かが変わりました。フラットデザインへの転換は美学的な革命のように語られますが、本当は宣言でもありました。「もうユーザーは、デジタルのものをリアルに見立てなくても理解できる」という。

それはある意味で、ケアの仕事が完成した瞬間でした。現実の比喩でコンピュータを説明する必要がなくなったのです。

## 道具から、関係へ

ここで問いが生まれます。次の振り子はどこへ向かうのか。

マテリアルデザインは美しい体系でした。Googleが2014年に提唱した「紙とインク」の比喩は、スキュアモーフとフラットの間に第三の道を切り開きました。物理法則を持ち込みながら、あくまでデジタルであるという誠実さ。でも一つの問いには答えられませんでした。AIという、従わない存在をどう扱うか。

これまでのUIデザインはすべて、道具は命令通りに動くという前提の上に成立していました。ゴミ箱アイコンは必ず削除する。ボタンは必ず反応する。だからデザインの仕事は「いかに正確に、わかりやすく操作させるか」でした。

AIはその前提を壊します。

同じ質問をしても毎回同じ答えが返るとは限らない。なぜその答えを出したかを説明できないこともある。時に間違える。それはまるで、猫と暮らすようなものです。完全には制御できないけれど、長く付き合うと、なんとなくわかってくる。完全にはわからないまま、信頼関係が育つ。

```
ハンマー  → 完全服従・意図通り動く
犬      → 訓練次第で従う・感情的な絆
猫      → 気まぐれ・でも関係が育つ
AI      → ← 今ここ
```

## 意図の解釈へ

デザインの重力が、静かに移動しています。

「どのボタンを押すか」という操作の設計から、「この人は今何をしたいのか」という意図の解釈へ。

それは一見、ユーザーにとって楽になる話のように聞こえます。言葉で伝えれば動く。操作の手順を覚えなくていい。でも実はここに、これまでのUIデザインが直面したことのない問いが隠れています。

AIが意図を誤解したとき、それをどう見せるか。どこまで先読みすると「便利」ではなく「気持ち悪い」になるか。間違えた提案が、会話の入り口になれるか。

## 画面の外へ

もう一歩進めると、問いはさらに広がります。

UIはずっと、画面の中にありました。スーザン・ケアが32×32ピクセルのキャンバスに向き合ったように、デザイナーは常に矩形の制約の中で考えてきました。スマートフォンになっても、タブレットになっても、その本質は変わらなかった。

でも今、その前提も揺らぎ始めています。

たとえば、服を選ぶという行為を想像してみてください。あなたがその日着ている服をAIが読み取り、リアルタイムの在庫の中から、あなたに似合う一点モノを提案する。その提案は、スマートフォンの画面ではなく、あなたがいる空間そのものに現れる。

そこにボタンはありません。メニューも、検索窓もない。あるのは、あなたと服と、その場の空気だけです。

ケアが「ゴミ箱」で解いたのは、デジタルを現実の言葉に翻訳するという問いでした。今起きていることは、その逆です。デジタルが現実の空間に溶け出し、翻訳そのものが必要なくなろうとしている。

## 意図の解釈へ

デザインの重力が、静かに移動しています。

「どのボタンを押すか」という操作の設計から、「この人は今何をしたいのか」という意図の解釈へ。

それは一見、ユーザーにとって楽になる話のように聞こえます。言葉で伝えれば動く。操作の手順を覚えなくていい。でも実はここに、これまでのUIデザインが直面したことのない問いが隠れています。

## 制御から対話へ

これまでのUIは、ユーザーが「主体」でシステムが「客体」でした。人間が命令し、機械が従う。デザインの仕事は、その命令をできるだけ摩擦なく届ける経路を作ることでした。

でも対話は違います。

対話には、二つの主体が必要です。どちらかが一方的に従うのではなく、互いが相手の言葉を受け取り、解釈し、応答する。その往復の中で、最初には存在しなかった何かが生まれる。

AIとのやりとりが面白いのは、まさにそこだと思っています。「こう聞いたらこう返ってくる」という予測が外れるとき、そこに発見がある。完璧に制御できないことが、むしろ豊かさになる。

デザインもそれに気づき始めています。ユーザーがシステムを操作するのではなく、ユーザーとシステムが共に場をつくる。そのための設計へと、問いの立て方が変わってきました。

```
操作のデザイン  →  ユーザーがシステムを制御する
対話のデザイン  →  ユーザーとシステムが共に場をつくる
```

難しいのは、対話には「正解の手順」がないことです。操作には正解があります。ファイルを削除したければゴミ箱に入れる、という一本道。でも対話は毎回違う。文脈が変わり、相手が変わり、その日の気分でさえ結果が変わる。

だからこそ、デザインは「完璧な操作体験をつくること」から離れ、「不完全な存在との関係が育てる場をつくること」へと向かっていくのだと思います。

## これから

スーザン・ケアがいなければ、マウスでアイコンをクリックするという「当たり前」は生まれなかった。では次の「当たり前」は何か。

おそらくそれは、洗練されたデザインシステムではなく、もっと不完全で、もっと人間に近いものになると思っています。間違えることがある。毎回同じ答えを返さない。でも、だから会話が生まれる。

制御から、関係へ。画面から、空間へ。操作から、意図へ。

振り子はまだ、揺れています。そしてその振り子の先に、自分がつくりたいものがあります。
